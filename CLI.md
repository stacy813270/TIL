## CLI

#### CLI란?

- GUI: 그래픽을 통해 컴퓨터와 상호작용 ex. 클릭

- CLI: 명령어를 통해 컴퓨터와 상호작용

  - 화면 상으로 안 보이기 때문에 경로 표기 필요
<br>

#### 문법 및 활용

- 디렉토리 = 폴더

- . ← 현재 디렉토리

- .. ← 현재의 상위 디렉토리 (cd .. ← 상위 디렉토리로 이동; 뒤로가기 버튼)

- touch: 파일 생성

- mkdir: 새 디렉토리 생성(make directory)

- ls: 현재 작업 중인 디렉토리 내부의 폴더/파일 목록을 출력(list)

- cd: 현재 작업중인 디렉토리를 변경 (위치 이동) ex. cd saffy/

- start: 폴더/파일을 열기

- rm: 파일 삭제(디렉토리 삭제는 -r 옵션을 추가 사용)

- pwd: 현재 작업 중인 디렉토리의 절대 경로를 출력

> ctrl + l:  모두 지우기

<br>

#### 경로

- 루트(root) 디렉토리(/): 모든 주소의 시작점
- 홈 디렉토리(~): 터미널을 처음 켰을 때 시작하는 나의 기본 공간
  - /c/Users/<사용자>/ (Window의 경우)

- 절대 경로: 루트부터 목적지까지의 전체 주소. 위치가 변하지 않는 설정 파일에 씀
- 상대 경로: ‘현재 내 위치’를 기준으로 한 주소. 일반적인 개발환경에서는 나를 기준으로 한 상대경로를 주로 씀
  ```
  # 현 위차에 속한 파일(ssafy)로 이동
  cd ./ssafy
  # 상위 파일로 이동
  cd ..
  ```

> **수업 내용을 정리하는 것에 더해서 추가적인 CLI를 AI한테 물어보고 공부하기. 교재에 있는 것은 기초를 위한 매우 일부분일 뿐.**
> 어디까지 공부하는지는 강사님께 여쭤보면 지금 할 수 있는 최선의 선택에 대해 알 수 있음
